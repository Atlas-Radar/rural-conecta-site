# Rural UI Quality

## When to use
Use for layout, components, CSS, responsive behavior, design tokens, interactions, dialogs, forms and visual reviews.

## Procedure
1. Validate mobile first.
2. Use semantic HTML and native CSS.
3. Keep touch targets comfortable.
4. Ensure focus and contrast.
5. Avoid heavy effects and permanent animations.
6. Do not put text inside images.
7. Use placeholders clearly when content is not confirmed.

## Verification
Report mobile viewport, desktop viewport, accessibility notes and visual risks.
